# 0012. 음악 플레이어 — YouTube 엔진 + 2겹(메뉴바 미니 + 창) 구조

- 상태: 채택
- 날짜: 2026-06-23

## 배경

데스크톱 OS 셸(ruehanix)에 음악 플레이어를 추가한다. 요구사항:
1. 재생 중에는 항상 들려야 한다.
2. "다른 탭으로 넘어가도" 재생이 끊기지 않아야 한다.
3. 사용자가 직접 큐레이션한 플레이리스트를 재생한다.

음원 소스와 미니플레이어 위치는 사전 합의로 각각 **YouTube**, **상단 메뉴바**로 결정했다.

## 결정

- **2겹 구조**: ① 셸 루트에 상주하는 숨긴 오디오 엔진, ② 상단 메뉴바 미니플레이어(항상 보임),
  ③ 풀 UI인 `music` 창 앱(`rhx-play`). 셋이 하나의 플레이어 상태(`useRuehanix`)를 공유한다.
  창을 닫아도 엔진은 살아 있어 재생이 유지된다(macOS 메뉴바 멘탈모델).
- **엔진 = YouTube IFrame Player API**. 화면 밖으로 숨긴 iframe 하나를 셸 루트에 두고
  `playVideo`/`pauseVideo`/`loadVideoById`/`setVolume`로 제어한다.
- **상태 분리**: 순수 로직(`lib/ruehanix/player.ts` reducer: next/prev/select/toggle/volume/repeat/onEnded)과
  영속화(`player-storage.ts`)는 엔진과 무관하게 단위 테스트한다. 엔진은 상태를 따라가는 부수효과 계층.
- **UI 낙관적 갱신**: 컨트롤 클릭 시 스토어 상태를 즉시 바꾸고 엔진이 뒤따른다. UI가 의도를 즉시 반영.

## 이유와 대안

- **YouTube vs mp3 자가호스팅**: 자가호스팅은 Media Session(미디어 키)·진짜 오디오 전용·ToS 무관이라 깔끔하나
  파일 호스팅 부담이 있다. YouTube는 호스팅 0 + 곡 무한이라 개인 포폴에 적합. 합의대로 YouTube 채택.
- **2겹 vs 단일 창**: 단일 창이면 창을 닫을 때 재생이 끊긴다. 메뉴바 미니로 분리해 "상시 재생" 요구를 만족.
- **셔플 보류**: 순수 reducer에 난수를 들이면 테스트 결정성이 깨진다. 반복(off/all/one)만 우선 구현,
  셔플은 백로그.

## 영향

- **재생 지속 범위 = 셸 내부**(앱 전환·워크스페이스 전환). 셸은 단일 컴포넌트라 앱 전환 시 언마운트되지 않아
  엔진이 그대로 유지된다 → 요구사항 2를 데스크톱 메타포 안에서 만족.
- **한계(v1 범위 밖)**: `/posts/[slug]` 등 풀 라우트로 이동하면 셸이 언마운트돼 재생이 멈춘다.
  진짜 크로스 라우트 지속은 엔진을 `app/layout.tsx`로 올려야 하며 별도 작업 — 백로그.
- **autoplay 정책**: 브라우저는 제스처 없는 자동재생을 막는다. 첫 재생은 사용자 클릭이 필요하고,
  재접속 복원 시 `playing`은 항상 false로 둔다(트랙·볼륨·반복모드만 복원).
- **모바일**: iOS Safari는 백그라운드 iframe 오디오를 강하게 제한 → 사실상 데스크톱 기능.
- **ToS**: 숨긴 백그라운드 재생은 YouTube 약관상 회색지대. 개인 포폴 맥락에서 위험 낮음으로 판단.
