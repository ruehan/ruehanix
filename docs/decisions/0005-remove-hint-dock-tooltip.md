# 0005. 첫 방문 힌트 제거, 독 호버 이름 라벨 추가

- 상태: 채택
- 날짜: 2026-06-22
- 관련: [[0004-app-discoverability]] (힌트 부분을 폐기)

## 배경

ADR 0004에서 발견성 보강으로 (1) 데스크톱 독, (2) 런처 버튼 강화, (3) 첫 방문 힌트(튜토리얼
말풍선)를 넣었다. 사용 후 사용자가 "튜토리얼 같은 힌트는 불필요, 제거"를 요청했고, 대신
독 아이콘에 마우스를 올리면 위에 앱 이름을 보여주는 호버 라벨을 원했다.

## 결정

1. **첫 방문 힌트 제거** — `AppsHint` 컴포넌트, `useRuehanix`의 hint 상태/9초 타이머/localStorage,
   `viewModel`의 showHint/dismissHint/openFromUi, `onboarding.ts`(+테스트), `@keyframes rh-pulse`를
   모두 삭제. 독·버튼 강화는 유지(발견성은 항상 보이는 독이 충분히 담당).
2. **독 호버 이름 라벨** — 데스크톱 독 아이콘 위에, 호버 시 앱 이름 pill을 CSS로 표시
   (`.rh-dock-label`, `.rh-dock-item:hover`로 opacity 전환). JS 상태 없이 순수 CSS.
   브라우저 기본 `title`은 제거하고 `aria-label`로 대체(스크린리더 접근성 + 중복 툴팁 방지).

## 이유와 대안

- **힌트 제거** — localStorage 1회 힌트는 첫 사용자 안내였으나, 항상 보이는 독이 이미 명확해
  튜토리얼이 군더더기라는 사용자 피드백. 유지보수 표면(타이머·localStorage·SSR 분기)도 줄어듦.
- **호버 라벨 CSS vs JS state** — 호버 표시는 순수 표현이라 React state가 불필요. CSS
  `:hover`가 가장 단순하고 리렌더 비용 0. 헬퍼/테스트도 필요 없음.
- **title → aria-label** — 커스텀 라벨이 시각 정보를 주므로 기본 `title` 툴팁은 중복.
  `aria-label`로 바꿔 스크린리더에 이름을 남기면서 시각 중복을 없앴다(0004 리뷰의 a11y 백로그 일부 해소).

## 영향

- 코드 순삭(net 감소). `DESKTOP_DOCK_RESERVE`·독 레이아웃은 그대로(독 유지).
- 스모크에 "독 호버 이름 라벨"(opacity 전환 완료 대기) 시나리오 추가 → 11개.
- 힌트가 사라져 첫 방문 발견성은 독·버튼에 전적으로 의존. 추후 필요하면 독 자체를 살짝
  강조하는 비침습적 방식으로 재고.
