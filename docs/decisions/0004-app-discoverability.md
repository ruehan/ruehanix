# 0004. 앱 실행 발견성 — 데스크톱 독 · 버튼 강화 · 첫 방문 힌트

- 상태: 채택
- 날짜: 2026-06-22
- 관련: [[0003-responsive-mobile-strategy]]

## 배경

데스크톱 진입점이 **Super+D(숨은 단축키)** 와 **좌상단 ruehanix 텍스트**뿐이라 일반 사용자가
"앱을 어떻게 여는지" 발견하기 어려웠다. 개발자에겐 충분하나 비개발자 발견성이 낮다.
세 가지를 묶어 보강한다: 항상 보이는 데스크톱 독 + 런처 버튼 강화 + 첫 방문 힌트.

## 결정

세 기능 모두 **데스크톱(≥768px) 한정**. 모바일은 이미 `MobileDock`이 있어 무변경.

1. **데스크톱 독** — 하단 중앙 floating pill, 8개 앱 아이콘. 클릭으로 앱 실행, 활성 앱 강조,
   호버 시 살짝 떠오름(`.rh-dock-item`). 타일 영역을 가리지 않도록 `area()`에 `bottomReserve`
   인자를 추가해 하단에 `DESKTOP_DOCK_RESERVE`(독 높이+마진)만큼 자리를 비운다.
2. **런처 버튼 강화** — 좌상단 버튼 아이콘을 사람 모양 → **9-dot 앱그리드**로 교체("앱 모음" 시그널),
   `title="앱 실행기 (Super + D)"` 툴팁 + 호버 강조(`.rh-launcher-btn`). 라벨 "ruehanix"·동작은 유지.
3. **첫 방문 힌트** — 부팅 후 독 위에 말풍선("여기서 앱을 열어보세요 · Super+D") + 펄스 링.
   `localStorage`(`rh-apps-hint-seen`)로 **1회만**. 9초 자동 사라짐, 또는 독·런처 클릭·말풍선 클릭 시 즉시 닫힘.
   표시 여부는 순수 함수 `shouldShowHint({seen,booting,isMobile})`로 결정(데스크톱·부팅완료·미열람).

## 이유와 대안

- **독 자리 확보(area reserve) vs floating 겹침** — 항상 보이는 독이 하단 타일 창을 가리면
  거슬린다. macOS처럼 공간을 확보(reserve)해 안 겹치게 했다. 순수 함수 변경이라 vitest로 커버,
  데스크톱 회귀가 아니라 의도된 레이아웃.
- **버튼 라벨 "Apps" vs "ruehanix"** — 브랜드 일관성을 위해 라벨은 "ruehanix" 유지하고,
  대신 9-dot 그리드 아이콘 + 툴팁으로 "클릭=앱 실행" 시그널을 줬다. 발견성은 독이 주로 담당.
- **힌트 강제 노출(런처 자동 오픈) vs 1회 힌트** — 자동 오픈은 매번 닫아야 해 거슬린다.
  localStorage 1회 힌트가 덜 침습적이면서 첫 사용자에게 충분.
- **기존 진입점 유지** — Super+D·waybar 버튼을 없애지 않고 보강만 했다(개발자 워크플로 보존).

## 영향

- `area(vp, gap, bottomReserve)` 시그니처 확장(기본 0이라 기존 호출 무영향). 데스크톱만 reserve 전달.
- 힌트 상태는 클라이언트 전용(localStorage·타이머) — SSR 기본 숨김(hintSeen=true)으로 하이드레이션 안전.
- 향후: 독 자동숨김 토글(Settings), 독 아이콘 재정렬/고정, 힌트 다국어.
